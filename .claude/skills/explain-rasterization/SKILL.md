### 2. 解释算法原理

#### 三角形光栅化基础
1. **目标**：将3D空间中的三角形转换为2D屏幕上的像素
2. **输入**：三个顶点坐标（经过透视除法后的标准化设备坐标）
3. **输出**：帧缓冲中的像素颜色

#### tinyrenderer中的实现步骤
1. **三角形设置**：
   - 计算三角形的2D包围盒（bounding box）
   - 确定需要遍历的像素范围

2. **像素遍历**：
   - 遍历包围盒内的每个像素
   - 计算像素的重心坐标（barycentric coordinates）

3. **内部测试**：
   - 检查重心坐标是否都在[0,1]范围内
   - 如果像素在三角形内部，则继续处理

4. **深度测试**：
   - 插值计算像素深度值
   - 与深度缓冲（zbuffer）比较
   - 如果更近，则更新深度缓冲

5. **着色计算**：
   - 调用片段着色器（fragment shader）
   - 计算最终像素颜色
   - 写入帧缓冲

### 3. 关键代码分析

#### 重心坐标计算
```cpp
// 重心坐标计算公式
// 给定三角形ABC和点P，计算α,β,γ
// α = area(PBC) / area(ABC)
// β = area(APC) / area(ABC)
// γ = area(ABP) / area(ABC)
// 其中 area(ABC) = (B-A) × (C-A) / 2
```

#### 光栅化核心循环
```cpp
// 伪代码示例：
for (int x = min_x; x <= max_x; x++) {
    for (int y = min_y; y <= max_y; y++) {
        vec3 bc = barycentric(triangle, x, y);
        if (bc.x < 0 || bc.y < 0 || bc.z < 0) continue;

        float z = interpolate_depth(triangle, bc);
        if (z < zbuffer[x + y*width]) {
            zbuffer[x + y*width] = z;
            TGAColor color = shader.fragment(bc);
            framebuffer.set(x, y, color);
        }
    }
}
```

### 4. 算法优化技术

#### 1. 包围盒优化
- 只遍历三角形实际覆盖的像素区域
- 减少不必要的像素计算

#### 2. 重心坐标优化
- 使用增量计算提高性能
- 避免重复计算面积

#### 3. 早期深度测试
- 在着色前进行深度测试
- 减少不必要的着色计算

### 5. 可视化解释

#### 光栅化过程示意图
```
      A
     / \
    /   \
   B-----C

像素遍历区域（包围盒）：
┌─────────────────┐
│                 │
│    △ ABC        │
│                 │
└─────────────────┘

内部像素测试：
○ 外部像素（跳过）
● 内部像素（处理）
```

#### 重心坐标可视化
- α,β,γ 表示点P相对于三角形ABC的位置
- 在三角形内部时：α,β,γ ≥ 0 且 α+β+γ = 1
- 在顶点A上时：α=1, β=0, γ=0
- 在边AB上时：γ=0

### 7. 常见问题解答

#### Q: 为什么有些三角形边缘有锯齿？
A: 这是光栅化的固有特性，可以使用抗锯齿（MSAA）技术缓解。

#### Q: 如何处理非常大的三角形？
A: 可以分块处理或使用层次包围盒优化。

#### Q: 重心坐标计算数值不稳定怎么办？
A: 使用双精度浮点数或改进的面积计算方法。

#### Q: 如何实现透明效果？
A: 需要修改深度测试和颜色混合逻辑。
